# DevOps

DevOps setzt sich zusammen aus "Development" und "Operations". Wie der Name schon vermuten lässt, sitzt es in der Mitte, zwischen den beiden Disziplinen. Die Aufgabe von DevOps besteht meines Verständnisses nach darin, einen robusten, sicheren, protokollierten und transparenten Prozess zu schaffen, die Entwicklung eines Produktes automatisiert auf die gewünschten Umgebungen (Staging, Test, Produktion, oder andere) zu bringen.

Das Ziel ist, dass Entwickler sich kaum damit beschäftigen müssen, was nach einem Push passiert und Admins keine manuellen Deployments durchführen oder begleiten müssen. Der Prozess läuft automatisiert und für beide Seiten transparent ab. Auch DevOps muss bei einer robusten Pipeline mit der Zeit nur noch in Ausnahmefällen eingreifen.
