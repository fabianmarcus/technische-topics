
# ARIA Attribute

## Was ist es?

ARIA (Accessible Rich Internet Applications) ist eine Spezifikation, die es ermöglicht, Webinhalte und Webanwendungen für Menschen mit Behinderungen zugänglicher zu machen.

ARIA-Attribute werden zu HTML-Elementen hinzugefügt, um zusätzliche Informationen über die Rolle, den Zustand und die Eigenschaften von Elementen bereitzustellen. Diese Informationen können von assistiven Technologien wie Screenreadern genutzt werden, um die Benutzererfahrung für Menschen mit Behinderungen zu verbessern.

ARIA Attribute sind aufgeteilt in zwei Kategorien:

- Roles
- States and Properties

## Roles

HTML-Attribute, die die semantische Funktion eines Elements im Dokument beschreiben.

Ein klar sehender Mensch kann visuell erkennen, dass ein Element eine Navigation, eine Animation oder eine Fortschrittsanzeige ist. Assistive Technologien können diese Informationen jedoch nicht immer korrekt interpretieren. Hier kommen die ARIA-Rollen ins Spiel.

Die Attribute sollten nur dann verwendet werden, wenn die semantische Bedeutung eines Elements nicht durch ein natives HTML-Element vermittelt werden kann. Ansonsten reicht das native Element aus.

### Widget Roles

So wird aus einem Div semantisch ein Button:

```html
<div role="button">Speichern</div>

<!-- Das native „button“-Element ist aber immer vorzuziehen! -->
 <button>Speichern</button>
```

So wird einer assistiven Technologie mitgeteilt, dass ein Element eine Fortschrittsanzeige ist:

```html
<div role="progressbar">10/20 / 10%</div>
```

Oder ein Dialog:

```html
<div role="dialog">Pop-Up Content</div>
```

### Document Structure Roles

*Document Structure Roles* beschreiben die Struktur eines Dokuments. Sie werden in der Regel von assistiven Technologien verwendet, um die Navigation innerhalb eines Dokuments zu erleichtern.

- grid
- toolbar
- textbox
- figure

```html
<div class="grid" role="grid">
  <div class="grid-row" role="row">
    <div role="gridcell">Zelle 1</div>
    <div role="gridcell">Zelle 2</div>
  </div>
</div>
<div role="toolbar">
  <button>Button 1</button>
  <button>Button 2</button>
</div>  
```

## States and Properties

*States and Properties* reichern ein Element mit weiteren, nicht sichtbaren Informationen an, um assistiven Technologien zu helfen, den Zustand und die Eigenschaften eines Elements zu verstehen (aktiviert, deaktiviert, ausgewählt, etc.).

Typische Beispiele:

- aria-checked
- aria-valuemax
- aria-readonly
- aria-busy

```html
<form action="post">
  <label for="firstName">First name:</label>
  <input id="firstName" type="text" aria-required="true" />
  <br>
  <label for="lastName">Last name:</label>
  <input id="lastName" type="text" aria-required="true" />
  <br>
  <label for="streetAddress">Street address:</label>
  <input id="streetAddress" type="text" />
</form>
```
